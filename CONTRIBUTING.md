# Introduction

Thank you for your interest in contributing!

> Before filing an issue or feature request, please have a cursory glance at others to see if a similar issue is already filed. 
> 
> It is not expected that every overlap will be caught, but some research is appreciated.

## How to report a bug
> If you find a security vulnerability, do NOT open an issue. Contact [@joelebutler](https://github.com/joelebutler) via one of the socials in his profile's bio for further support.

When filing an issue, make sure to answer these five questions:

1. What version of CSICK are you using (csick version)?
2. What operating system and processor architecture are you using?
3. What did you do?
4. What did you expect to see?
5. What did you see instead?

## How to suggest a feature or enhancement

To request a new feature or enhancement, open an issue with a detailed description of your request. Be responsive in requests for more information, and try to be as proactive as possible in addressing issues that may come up.

## Making a Code Contribution

1.  Make a personal fork of the repo if you have not already.
2.  Make a branch for the issue with the following format
    ```
    XXXXX-my-branch-name
    ```
3.  Fix the issue or fulfill the request.
4.  Ensure all tests pass.
5.  Make a commit to your branch with the following format and push
    ```
    #XXXXX: My brief synopsis of changes.
    ```
6. On the issue page, a button will appear to make a pull request. Please do so describing any changes you made. 
7. I will make an effort to review this as soon as possible to keep up-to-date with all fixes.
